Uni-Run
꼭 오브젝트의 전체가 아닌 닿는지 판정하는 부분만 Collider를 해도 되겠다라는 생각

Animator - Has Exit Time - 종료 시점 Exit Time을 활성화하는 옵션
	 - Transition Duration - 전환 지속 시간은 전이가 이루어지는 동안 현재 애니메이션 클립과 다음 애니메이션 클립을 섞어(블렌딩) 부드럽게 이어주는 역할

배경을 삽입할 때 레이어 정렬이 필요한데, Sprite Renderer - Sorting Layer - Add Sorting Layer하여 Layer 층을 구성하여 각 오브젝트에 할당

계속 반복되는 배경을 만들어야하기 때문에 배경 뒤에 똑같은 배경을 붙히는 방식으로(범위 밖으로 지나간 배경은 다시 뒷 배경의 뒤로 이동)
* 기본적으로 vector2는 잘 사용하지않고 vector3를 일반적으로 사용

다양한 크기와 비율의 화면에서도 UI의 크기와 배치를 일정하게 유지하고자 하면
기준 화면 크기를 정하고, 실행 화면이 기준 화면보다 크거나 작을 때는 자동으로 확대/축소(스케일업/스케일다운)
하는 화면 크기에 따라 스케일 모드를 UI 스케일모드로 사용할 수 있음
UI - Canvas - UI Scale Mode - Scale With Screen Size로 변경

게임 매니저처럼 관리자 역할을 하는 오브젝트(예를 들면 파일 매니저, 몬스터 매니저, 점수 등)는 일반적으로
프로그램에 단 하나만 존재해야 함(단일 오브젝트). 그리고 언제 어디서든 즉시 접근 가능해야 함.(손쉬운 접근)

컴포넌트가 활성화될 때 해당 컴포넌트의 OnEnable() 메소드 실행
